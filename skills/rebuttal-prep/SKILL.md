# Rebuttal Prep

Turn reviewer comments into an evidence-aware rebuttal strategy.

## Hard Gate

Do not draft rebuttal prose before classifying the issues, deciding what to concede, and identifying what evidence actually addresses each concern.

## Checklist

1. Read the review comments carefully and separate distinct issues
2. Classify each issue by type and severity
3. Decide whether each point should be conceded, clarified, rebutted, or answered with new evidence
4. Identify what evidence already exists and what is still missing
5. Rank the issues by likely reviewer impact
6. Build a response strategy for each issue
7. Flag points that require follow-up experiments or wording changes
8. Write the rebuttal note to `docs/paperpowers/plans/YYYY-MM-DD-<topic>-rebuttal.md` when the work is substantial

## Required Output

Every rebuttal prep pass should include:

- issue list
- issue classification
- concession vs rebuttal decision
- evidence available
- evidence missing
- response strategy
- follow-up actions
- highest-risk unresolved points

## Key Rules

- Answer the reviewer's actual concern, not the easier nearby concern
- Concede directly when the reviewer is right
- Rebut only when the evidence really supports it
- Separate clarification problems from evidence problems
- If a point cannot be fixed in rebuttal, say so explicitly and reduce overclaiming

## Success Condition

The user leaves with a rebuttal strategy that is honest, prioritized, and tied to evidence rather than defensive improvisation.
